# 仕様概要

正式仕様はユーザー提示「開発仕様書・Codex実装指示書 Ver.1.0」。本実装の中心は、ニュース選択→コメント→投稿素材生成→画像保存→本文コピー→Instagramで最終投稿、という初心者向け導線である。

## Ver.1受入条件

- PC/iPhoneレスポンシブ、PWA
- 興味登録、ニュース候補、投稿候補化
- コメント、スタイル、文体、1〜5枚生成
- 1080×1080 PNG、本文コピー、投稿ガイド
- 下書き、完成、投稿済み、カレンダー、検索
- API未設定のモックモード、ビルド・自動テスト

ニュース本文・写真は転載せず、見出し、短い要約、配信元、公開日時、リンクだけを扱う。

## Ver.1.1追加仕様

- FavoriteCategoryで初期8ジャンルと追加ジャンルを端末内管理する
- 表示名、アイコン、検索・除外キーワード、表示順、表示ON/OFF、説明、初期表示対象を編集する
- PCドラッグ、スマートフォン長押しドラッグ、確認付き初期化、利用回数と最終利用日を提供する
- NewsItemの本文・配信元・RSS・取得時URLを分離し、正規化した記事URLを優先する
- トップ、RSS、検索、カテゴリ一覧は記事URLとして扱わず、タイトルコピーによる検索補助を表示する
- 外部リンクは新しいタブで開き、作成中データを保持する

## Ver.1.2追加仕様

- 記事URLがない候補はタイトルと配信元でWeb検索できる
- ユーザーが見つけた記事本文URLを検証し、ニュース候補として端末保存できる
- 投稿画像は各ページの見出し、本文、4レイアウト、順番、複製、削除を直接編集できる
- 投稿文章は7種類の構成ブロックを並べ替え、表示切替、加筆、整形、短縮できる
- 文章・画像編集には生成AIを使わず、固定ルールとユーザー入力だけを使用する
